CSS Grid

  • Don't forget your name! And what you do.
  • Intro rather than a lesson
    • Assuming most are familiar with flexbox
    • Flexbox defines a 1-dimensional system - either vertical or horizontal
    • CSS Grid is a new-ish set of properties; defines a 2-dimensional grid-based layout system

    See the Pen Flex and Grid demo by rachelandrew (@rachelandrew) on CodePen.

    Rachel Andrew's Article: https://rachelandrew.co.uk/archives/2016/03/30/should-i-use-grid-or-flexbox/ Rachel Andrew's Pen: https://codepen.io/rachelandrew/pen/YqqdXL My fork: https://codepen.io/arelia/pen/WMKmJX/?editors=1100
    • Two sets of cards. Both in rows
    • Flexbox - Resize page and cards go to next row. If you allow them to grow and shrink, they fill the space in the row
    • Grid - Same but vertical relationship as well

    Like Flexbox, Grid is good for

    • Dynamic content
    • Responsive layouts
    • Uniform height/width across elements
    • Changing order
  • Create a layout and position elements where you want them

    • Fallbacks with @supports

    For production, if it doesn't "just work" in older browsers, use fallbacks like display: flex or display: table, or do nothing b/c block elements will still be block elements
